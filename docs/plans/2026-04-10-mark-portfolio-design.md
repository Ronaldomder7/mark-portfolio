## 1. 项目宗旨

这不是一份求职作品集，是一个**"让人窥见马克世界"的数字门厅**。

成功标准（按优先级）：
1. **C · 长期资产（主）** — 活着的数字自我，陪马克很多年
2. **A + B · 副产品** — 招聘机会 + 有趣的人际连接

**核心约束**：马克**不想自己往里填内容**，希望网站能从 Obsidian + flomo 里汲取"最新的他"。

**灵魂金句**（罗振宇）：
> 人的魅力，就是拥有一个更大的但别人进不去的世界。

---

## 2. 系统架构

这个项目不是"一个网站"，是**三层系统**：

```
数据源层  →  中间层（扫描/审核）  →  展示层（Next.js 网站）
↑                                              │
└───── Mark 照常在 Obsidian/flomo 写笔记 ──────┘
```

**Mark 的操作面只有两个命令**：
```bash
npm run sync     # 扫描 Obsidian/flomo，生成候选文件到 Obsidian
npm run publish  # 把勾选后的候选发布到网站
```

## 7. 视觉设计

| 维度 | 决策 |
|---|---|
| **布局** | 单页长滚动 + 顶部极简锚点导航（门面/作品/思想/底色/生长） |
| **背景** | 暖白 `#FAFAF8` |
| **正文** | Noto Serif SC（思源宋体） |
| **强调** | Noto Sans SC Regular |
| **主色** | 纯黑 `#1A1A1A` |
| **点缀** | 深红 `#8B2E2E`（极少量） |
| **留白** | 每个板块之间留 1 屏呼吸 |
| **动画** | 滚动淡入，无其他 |

**5 层布局**：
1. **门面** · 只有 hook + 向下箭头，无头像无 CTA
2. **作品** · 4 个数据卡 + 3 张代表作品缩略图（申请制）
3. **思想世界** · 2 句金句 + 2 段思想纹理 + 知识系统证据卡
4. **生命底色** · 4 节点时间线
5. **正在生长** · 最近在想 4 条（由 sync/publish 脚本动态更新）

**页脚**：联系方式 + 一句"我缺的不是技能，是让独特性被看见的放大器" + 版本号 + 最后更新日期

## 10. 数据流 · npm run sync

```
1. 读取 scripts/config.js
2. 遍历每个数据源，按策略抽取候选
3. 禁区词过滤（命中任一则丢弃整段）
4. 时间窗口过滤（30 天）
5. 去重 + 按日期倒序
6. 生成候选文件到 Obsidian/.../网站候选/网站候选_YYYY-MM-DD.md
```

## 11. 数据流 · npm run publish

```
1. 读取候选文件
2. 解析 [x] 勾选项
3. 写入 content/recent.json
4. git add + commit + push
5. Vercel 自动重新部署
```

## 13. 风险与应对

| 风险 | 应对 |
|---|---|
| 禁区词漏检（误放私密内容） | 候选文件给马克二次审核，最终控制权在他 |
| flomo 文件名编码问题 | 扫描时按 date 前缀 parsing，忽略文件名中文内容 |
| Claude 对话存档 1039 文件扫描慢 | 白名单文件名过滤优先，减少 90% 文件数 |
| 构建失败阻塞发布 | CI 日志 + 本地 dev 测试优先 |
| Vercel 免费额度超限 | 不太可能（个人站流量极低） |
